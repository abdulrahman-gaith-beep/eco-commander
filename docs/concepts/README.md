# Concepts

Understanding-oriented explanations of eco-commander's core ideas. Read these
pages when you want to understand *why* the system is built a certain way —
not how to configure it or what a specific command does.

## In this directory

| Document | What it explains |
|---|---|
| [mental-model.md](./mental-model.md) | The operator console model, snapshots, meters, recipes, the widget vs. CLI router, the scheduler's role, and the trust boundary — as one coherent story |

## Diataxis note

The docs in this directory are **explanation** in the
[Diataxis](https://diataxis.fr/) sense: they are understanding-oriented and
discussion-heavy. They are not tutorials (step-by-step tasks for beginners),
how-to guides (goal-oriented tasks for practitioners), or reference (precise
information retrieval). If you need those, use the links below.
